# level-achievement-system.md — MMORPG 레벨 & 업적 시스템

모든 디자인 작업(Case A/B/C/W) 완료 시 **recording** 단계에서 자동 실행된다.
design-evaluator가 3축 평가 직후 경험치 부여 → 레벨업 판정 → 업적 갱신을 수행한다.

---

## 1. 트리거

```
design-evaluating 단계 완료 → recording 단계 진입 시:
  1. 3축 평가 점수 확정
  2. 경험치 계산 & 부여
  3. 레벨업 판정
  4. 업적 갱신
  5. 결과를 사용자에게 RPG 스타일로 출력
```

## 2. 경험치(XP) 획득 공식

```
획득XP = 기본XP × 등급배율 × 케이스배율 × 파이프라인배율

기본XP = 평가점수 × 10
  예) 85점 → 850 기본XP

등급배율 (design-craft.md 등급 기준):
  A등급 (80~100점): ×5  — 유니크 몬스터
  B등급 (60~79점):  ×3  — 챔피언 몬스터
  C등급 (40~59점):  ×1  — 일반 몬스터
  D등급 (0~39점):   ×0.5 — 약한 몬스터

케이스배율:
  Case A (Template):  ×1.2  — 리서치 탐험 보너스
  Case B (Project):   ×1.2  — 프로젝트 설계 보너스
  Case C (Copycat):   ×1.2  — 분석 보너스
  Case W (Web):       ×1.2  — 구현 보너스

파이프라인배율 (harness.config.json 참조):
  A→B:     ×1.2  (양쪽 60점+)
  A→W:     ×1.2  (양쪽 60점+)
  B→W:     ×1.3  (양쪽 60점+)
  C→W:     ×1.3  (양쪽 60점+)
  C→B:     ×1.2  (양쪽 60점+)
  A→B→W:   ×1.5  (전체 60점+)
  C→B→W:   ×1.5  (전체 60점+)
```

### 계산 예시

| 상황 | 점수 | 등급 | Case | 파이프라인 | 기본XP | 등급배율 | 케이스배율 | 파이프라인 | **최종XP** |
|------|------|------|------|-----------|--------|---------|----------|-----------|-----------|
| CAT13 신규 추가 | 93 | A | A | 없음 | 930 | ×5 | ×1.2 | ×1.0 | **5,580** |
| blumn.ai 분석 | 94 | A | C | 없음 | 940 | ×5 | ×1.2 | ×1.0 | **5,640** |
| AI컴포넌트 HTML | 90 | A | W | C→W | 900 | ×5 | ×1.2 | ×1.3 | **7,020** |
| Cyberpunk HUD | 92 | A | W | A→W | 920 | ×5 | ×1.2 | ×1.2 | **6,624** |
| 퍼블리셔 앱 | 73 | B | B | 없음 | 730 | ×3 | ×1.2 | ×1.0 | **2,628** |
| 저점수 D등급 | 30 | D | W | 없음 | 300 | ×0.5 | ×1.2 | ×1.0 | **180** |

## 3. 레벨 시스템 (Lv.1 ~ Lv.100)

### 3.1 필요 경험치 공식

레벨 L에서 L+1로 올리는 데 필요한 XP:

```
구간별 필요XP(L → L+1):

Lv. 1~10  (견습 디자이너):    XP(L) = 100 + 50 × L
Lv. 11~50 (숙련 디자이너):    XP(L) = 300 + 100 × (L - 10)
Lv. 51~70 (전문 디자이너):    XP(L) = 4,300 + 300 × (L - 50)
Lv. 71~90 (마스터 디자이너):  XP(L) = 10,300 + 800 × (L - 70)
Lv. 91~99 (전설의 디자이너):  XP(L) = 26,300 + 3,000 × (L - 90)
```

### 3.2 주요 레벨 필요 XP 참조표

| 레벨 | 다음 레벨 필요XP | 누적 총 XP (대략) | 칭호 |
|------|----------------|-----------------|------|
| 1→2 | 150 | 150 | 펜슬을 잡은 모험가 |
| 5→6 | 350 | 1,250 | 레이어를 아는 자 |
| 10→11 | 600 | 3,150 | 견습 디자이너 |
| 25→26 | 1,800 | 18,900 | 프레임 워리어 |
| 50→51 | 4,300 | 92,850 | 숙련 디자이너 |
| 70→71 | 10,300 | 231,550 | 전문 디자이너 |
| 80→81 | 18,300 | 374,550 | 모션 마에스트로 |
| 90→91 | 26,300 | 558,550 | 마스터 디자이너 |
| 95→96 | 41,300 | 727,550 | 픽셀의 현자 |
| 99→100 | 53,300 | 916,750 | 전설의 디자이너 |

### 3.3 레벨 구간별 칭호

| 구간 | 칭호 | 설명 |
|------|------|------|
| 1~5 | 펜슬을 잡은 모험가 | 디자인 세계에 첫 발을 내딛다 |
| 6~10 | 레이어를 아는 자 | 이제 좀 감이 오기 시작 |
| 11~25 | 프레임 워리어 | 꾸준히 디자인을 생산하는 전사 |
| 26~50 | 숙련 디자이너 | 하네스가 몸에 배기 시작 |
| 51~70 | 전문 디자이너 | 3축 평가가 두렵지 않은 경지 |
| 71~80 | 모션 마에스트로 | 어떤 애니메이션이든 자유자재 |
| 81~90 | 마스터 디자이너 | 하네스를 역으로 가르치는 자 |
| 91~95 | 픽셀의 현자 | 한 프레임에 경험치가 녹아든다 |
| 96~99 | 전설 그 자체 | 더 이상 증명할 것이 없는 자 |
| 100 | Pencil Creator의 신화 | 만렙. 당신이 곧 하네스다 |

## 4. 업적(Achievement) 시스템

### 4.1 업적 유형

#### A. 디자인 카테고리 최초 도전 업적

디자인 작업을 카테고리로 분류하고, **해당 카테고리의 첫 작업 완료 시** 업적 달성.

| 카테고리 | 첫 도전 업적명 | 패러디 원본 |
|----------|--------------|-------------|
| UI컴포넌트 | "나는 배치한다 고로 존재한다" | 데카르트 "나는 생각한다 고로 존재한다" |
| 애니메이션 | "움직이는 건 다 내 거야" | 라이온킹 "빛이 닿는 건 다 내 거야" |
| 데이터시각화 | "해저 2만픽셀" | 쥘 베른 「해저 2만리」 |
| 인터랙션 | "클릭의 연금술사" | 하가렌 「강철의 연금술사」 |
| 배경효과 | "뒤에서 빛나는 자" | - |
| HTML구현 | "빛으로 렌더얍!" | 세일러문 변신 대사 |
| 사이버/디지털 | "네온사인은 꺼지지 않아" | 웹툰 「달빛조각사」 패러디 |
| 자연/파티클 | "바람과 함께 Float" | 영화 「바람과 함께 사라지다」 |
| 음성/웨이브 | "주파수가 들린다" | 영화 「주라기 공원」 패러디 |
| 프로젝트설계 | "집밥 아키텍트" | tvN 「집밥 백선생」 |

#### B. 카테고리 반복 업적

동일 카테고리에서 디자인을 반복 작성하면 단계별 업적 달성:

| 횟수 | 업적 접미사 | 예시 (애니메이션 카테고리) |
|------|-----------|----------------------|
| 5회 | "~의 단골손님" | "애니메이션의 단골손님" |
| 15회 | "~에 미친 자" | "애니메이션에 미친 자" |
| 30회 | "~을 지배하는 자" | "애니메이션을 지배하는 자" |

#### C. 특별 업적 (숨겨진 업적)

| 조건 | 업적명 | 패러디 |
|------|--------|--------|
| 첫 디자인 완료 | "펜슬을 잡은 순간" | - |
| A등급 첫 달성 | "넌 내게 A를 줬어" | "넌 내게 반했어" |
| 3축 만점 (100점) | "가슴이 웅장해진다" | 인터넷 밈 |
| 파이프라인 보너스 첫 달성 | "콤보 개시!" | 격투게임 콤보 |
| 연속 3회 A등급 | "유혹을 이겨낸 자" | 마비노기 타이틀 |
| 연속 5회 A등급 | "A는 멈추지 않아" | "정의는 멈추지 않아" |
| 5개 케이스 타입 모두 경험 | "만능 디자이너" | 마비노기 타이틀 |
| COMBINED SAMPLE 첫 생성 | "합체 로봇" | 슈퍼로봇 대전 |
| 8개 디자인 카테고리 달성 | "슈퍼스타" | 마비노기 타이틀 |
| A→B→W 완전 파이프라인 | "풀콤보 달성!" | 리듬게임 |
| 레벨 50 도달 | "참을 수 없는 디자인" | 쿤데라 소설 패러디 |
| 레벨 100 도달 | "Pencil Creator의 신화가 된 자" | - |

  ████████░░░░░░░░ 2,450 / 4,500 XP

  다음 레벨까지: 2,050 XP (A등급 디자인 약 1회)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 7. 디자인 카테고리 판별 규칙

디자인 주제를 아래 키워드로 카테고리 분류:

| 카테고리 | 판별 키워드 |
|----------|-----------|
| UI컴포넌트 | 버튼, 카드, 폼, 입력, 사이드바, 네비, 모달, 테이블 |
| 애니메이션 | 트랜지션, 이징, 키프레임, 스토리보드, 모션, 페이드, 스케일 |
| 데이터시각화 | 차트, 그래프, 대시보드, 통계, 데이터바인딩 |
| 인터랙션 | 클릭, 호버, 드래그, 스크롤, 제스처, 인터랙티브 |
| 배경효과 | 파티클, 그라데이션, 블러, 글로우, 배경, 오버레이 |
| HTML구현 | HTML, CSS, JavaScript, 웹, 반응형, SVG, Canvas, WAAPI |
| 사이버/디지털 | 글리치, 네온, 홀로그램, 매트릭스, 스캔라인, 사이버펑크, HUD |
| 자연/파티클 | 꽃잎, 눈, 비, 바람, 계절, 자연, 파티클 시스템 |
| 음성/웨이브 | 음성, 파동, 웨이브, 오디오, 주파수, 스펙트럼 |
| 프로젝트설계 | 앱 디자인, 화면 설계, UX 흐름, 와이어프레임, 프로토타입 |
