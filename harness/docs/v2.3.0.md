# v2.3.0 — Case C (Copycat) 워크플로우 추가

**날짜**: 2026-03-29
**변경 유형**: 마이너 (새 워크플로우 추가)

---

## 변경 동기

blumn.ai AIVY AI Agent 섹션의 통화음성 Lottie 애니메이션을 WebFetch + Playwright로 분석하여
JSON 기법 정의서를 성공적으로 생성한 세션에서 검증된 워크플로우를 정식 Case로 편입.

기존 Case A(WPF→XAML 확보)와 달리, Case C는:
- **소스가 WPF가 아닌 실제 웹사이트** (Lottie, CSS, GSAP 등 웹 네이티브 기술)
- **XAML이 아닌 JSON으로 기법 정의** (design/json/sample/ 하위)
- **JSON 기반으로 펜슬 컴포넌트 축적** (재활용 목적 동일)

---

## 변경 내용

### Layer 1 — Knowledge (design-craft.md)
- Case C 평가 축 3개 추가:
  - **C1 (35점)**: 조사 깊이 & 정확성 — WebFetch+Playwright 병행, 구현 기술 식별, Lottie 파싱
  - **C2 (35점)**: JSON 구조 완결성 — 기법별 독립 JSON, 필수 필드, 재현 코드
  - **C3 (30점)**: 펜슬 컴포넌트 품질 — JSON↔카드 매핑, Before→After, 재활용성
- 파이프라인 추가: C→W (1.3x), C→B (1.2x), C→B→W (1.5x)

### Layer 2 — Agents (design-evaluator.md)
- 기존 로직 유지 (Case 레터 자동 판별)

### Layer 3 — Engine
- **design-journey.md**: Case C 상태 전이 경로 추가, Pipeline C→W/C→B→W 추가
- **harness.config.json**: v2.3.0 범프, Case C 정의, 파이프라인 보너스, type_multiplier 추가

### Skills
