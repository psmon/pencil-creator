# Harness v2.2.0 — .pen + .xaml 이중 참조 필수화

- 날짜: 2026-03-29
- 버전: 2.1.0 → 2.2.0 (마이너)
- 트리거: Case B/W에서 .pen만 참조하면 퀄리티가 떨어지는 문제 발견

---

## 핵심 발견

WPF 애니메이션을 웹으로 구현할 때, **두 가지 소스**가 각각 다른 정보를 제공한다:

| 소스 | 제공 정보 | 단독 참조 시 결과 |
|------|----------|-----------------|
| `.pen` (wpf-animation.pen) | 시각 구조, 카드 레이아웃, 색상 체계, Before→After | "대략적" 구현 — 어떤 기법인지는 알지만 수치가 부정확 |
| `.xaml` (design/xaml/sample/*.xaml) | Duration, EasingFunction, From/To, BeginTime 등 정밀 파라미터 | "정확한" 구현 — WPF 원본과 동일한 타이밍/이징 매핑 가능 |

**.pen만 읽으면**: "QuadraticEase로 떨어진다" 수준 → 임의 easing 적용
**.pen + .xaml 둘 다 읽으면**: "Duration 4s, QuadraticEase EaseIn, Y: 0→650" → 정밀 매핑
