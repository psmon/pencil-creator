## Case B 세션 경험 기반 (2026-03-28 publisher-app)

### 성공 패턴
1. **Phase 2a 정적 디자인**: 스타일 가이드(Soft Bento + Tangerine Orbit) 로딩 → 재사용 sidebar 컴포넌트 → 4개 화면 순차 구축
2. **Phase 2b 애니메이션 가이드**: 동일 .pen 파일 하단에 별도 섹션 → 화면별 CAT-A~D → 각 3개 카드 (총 12개)
3. **룩앤필 매칭**: 처음 다크 테마로 시작 → 사용자 피드백으로 밝은 테마로 전환 (디자인 상식!)
4. **wpf-animation.pen 참고**: CAT1(Input), CAT2(Feedback), CAT3(Navigation), CAT4(Decorative), CAT5(3D Transform), CAT7(Sequential), CAT8(Interactive), CAT9(Data Viz) 총 8개 카테고리에서 기법 차용

### 학습 포인트
- 정적 디자인과 애니메이션 가이드는 반드시 **같은 룩앤필** 유지
- 애니메이션 가이드 카드에 **Target 프레임 경로** 명시가 핵심
- wpf-animation.pen은 **직접 수정하지 않고 참고만** 함 (Case A와의 구분점)
