# Harness v2.1.0 — WPF→Web 구현 품질 상향 업데이트

- 날짜: 2026-03-28
- 버전: 2.0.0 → 2.1.0 (마이너)
- 트리거: 봄 애니메이션 플레이그라운드 Case W 피드백 반영 후 학습사항 반영

---

## 변경 사항

### Layer 1: knowledge/design-craft.md

#### Case W 축 W2 (애니메이션 충실도) 상향
- **35점 상한 기준 추가**: 다중 CAT 활용(4개+) + WAAPI 순차 시퀀스 + SVG 리얼리티
- **WPF→Web 매핑 테이블 확장**:
  - QuadraticEase → JS quadratic easing
  - SineEase → CSS ease-in-out
  - ElasticEase → cubic-bezier(0.175, 0.885, 0.32, 1.275)
  - Path stroke animation → SVG stroke-dasharray/dashoffset
  - DropShadowEffect → filter: drop-shadow()
  - Confetti Burst → WAAPI particle system
- **SVG 리얼리티 판정 기준 신설**:
  - radialGradient 적용 여부 (flat fill = 감점)
  - 통합 SVG 구조 여부 (분리 = 감점)
  - 초기 fade-in 처리 여부

#### Case W 축 W3 (독창적 확장) 상향
- **30점 상한 기준 추가**: 내러티브 시퀀스 애니메이션 + 파라미터 조절 UI + 외부 이미지 불필요
- **판정 방법 확장**: 내러티브 시퀀스, 파라미터 제어, 자체 리소스 생성

#### 파이프라인 보너스 추가
- **A → W**: 양쪽 60점+ → 각 XP × 1.2 (직통 파이프라인)

### Layer 2: harness-usage 스킬

#### Case W Phase 2 전면 개편
- XAML → Web 매핑 테이블 12개 항목으로 확장
- **SVG 리얼리티 필수 규칙 4조항** 신설:
  1. 자연물에 radialGradient 적용 (flat fill 금지)
  2. 복합 요소는 통합 SVG 1개로 구성
  3. Canvas 파티클 초기 fade-in 적용
  4. 외부 이미지 대신 SVG Blob URL 자체 생성
- **순차 시퀀스 애니메이션** 가이드 추가 (WAAPI Promise chain)
- **파라미터 제어 UI** 의무화 (슬라이더 제공)
- **다중 CAT 활용** 의무화 (4개+ 카테고리)
- Pipeline 보너스 테이블에 A → W 추가

### Layer 3: harness.config.json
- version: 2.0.0 → 2.1.0
- Case W label: "Pencil → HTML (SVG Realism + WAAPI)"
- pipeline_bonus에 A_W 추가
