# v2.6.0 — Case D 추가 (DesignMD → Pencil 디자인 시스템 복제)

**날짜**: 2026-04-17
**버전 범프**: 2.5.1 → 2.6.0 (마이너 — 새 평가 케이스 추가)

---

## 변경 동기

외부 디자인 시스템 마켓(`designmd.ai`)의 CLI(v0.2.0)로 **DESIGN.md 파일을 영입**하여,
`design/design-md.pen`에 **프레임 단위로 재구성**하는 워크플로우가 새롭게 요구됨.

- 기존 Case B는 내부 자원(wpf-animation.pen)을 참고하는 디자인 흐름
- Case D는 **외부 디자인 시스템 영입 → 펜슬 재구성**의 별도 라인
- TOKENS · COMPONENTS · TEMPLATES 3계층 프레임화로 이후 Case B/W에 즉시 재사용 가능한 라이브러리 축적

- "designmd에서 OO 디자인 영입해줘"
- "디자인엠디 디자인 시스템 복제"

### 핵심 CLI (Bash)
```
designmd search "<키워드>" --limit 5
designmd get <user/slug>
designmd download <user/slug> -o design/design-md/<slug>/DESIGN.md
```

## 변경 내역

### Layer 1 — Knowledge (design-craft.md)
- 1절 Case 표에 **Case D 행** 추가
- **4-2절 Case D: DesignMD → 펜슬 디자인 시스템 복제** 신규 섹션 (D1/D2/D3 기준·판정법)
- 7절 파이프라인 평가에 **D→B / D→W / D→B→W** 항목 추가
- 8절 평가 절차에 Case D 트리거·컨텍스트 수집 규칙 추가

### Layer 2 — Agents (design-evaluator.md)
- 역할 정의: 입력 항목에 DESIGN.md 원본 추가, 출력을 `A/B/C/D/W`로 확장
- 평가 기준: **Case D 3축 명세** 추가
- 결과 출력 형식: `Case {A|B|C|D|W}`로 확장
- 파이프라인 감지: `D→B (×1.3), D→W (×1.3), D→B→W (×1.5)` 추가
- Recording 단계: 로그 파일명에 `case{A|B|C|D|W}` 반영

### Layer 3 — Engine (design-journey.md)
- 상태 다이어그램: Case A/B/C/**D**/W 유형 판별 추가
- researching/designing 상태 설명에 Case D 행동 추가
- **Case D 경로** 추가: `researching(designmd 수집) → designing(design-md.pen 프레임 복제)`
- **Pipeline D→B→W**, **Pipeline D→W** 경로 신규 추가
- 상태별 담당 표에 Case D 행동 반영

## 네임스페이스 주의

문자 `D`가 두 문맥에서 사용됨 — **스코프가 다르므로 런타임 충돌은 없음**:

| 키 | 의미 |
|----|------|
| `evaluation.grades.D` / `rpg_system.xp_formula.grade_multiplier.D` | **Grade D** (0-39점, 최하위 등급) |
| `evaluation.design.cases.D` / `rpg_system.xp_formula.type_multiplier.D` | **Case D** (DesignMD Replication) |

로그 파일명 `case{A|B|C|D|W}.md`의 `D`는 항상 Case D를 의미.
