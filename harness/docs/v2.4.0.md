# v2.4.0 — COMBINED SAMPLE 단계 도입

**날짜**: 2026-03-29
**유형**: 마이너 (워크플로우 변경)
**트리거**: blumn.ai 원형 음성파동 분석 Case C에서 통합 컴포넌트 프레임의 가치 발견

---

## 변경 사유

개별 컴포넌트 카드(CAT별 분산)는 기법 학습과 자유 조합에 유리하지만,
실제 웹사이트에서 발견한 애니메이션은 **여러 기법이 조합된 완성체**인 경우가 대부분이다.

예) blumn.ai 음성파동 = Radial Layout + Trim Path Pulse + Color Cascade 3기법 조합

사용자가 이 애니메이션을 구현하려면 3개 카드를 각각 찾아 머릿속에서 조합해야 하는데,
**COMBINED SAMPLE 프레임**이 있으면 한 프레임만 참조하여 즉시 구현 가능하다.

### COMBINED SAMPLE 프레임 구성 표준

```
SAMPLE — {이름} (통합 컴포넌트)
├── [COMBINED SAMPLE] 배지 (#E4324F) + Source URL
├── 타이틀 (한영 병기) + 기법 파이프라인 요약
├── content-row (horizontal)
│   ├── VISUAL STRUCTURE (좌측)
│   │   ├── 실제 동작의 정적 표현 (radial lines, particles 등)
│   │   └── 범례 (Active / Inactive / Transition)
│   └── ANIMATION SPEC (우측)
│       ├── 핵심 수치 그리드 (Canvas, Lines, Trim, Color, Stagger, Easing)
│       └── JSON/XAML REFERENCES (파일 경로 ← 기법명)
└── unified-code (horizontal)
    ├── WPF IMPLEMENTATION (코드 블록)
    └── CSS/SVG IMPLEMENTATION (코드 블록)
```

### 3. 개별 카드 vs COMBINED SAMPLE 역할 분리

| 산출물 | 역할 | 사용 시점 |
|--------|------|----------|
| 개별 카드 (CAT별) | 기법 단위 학습, 자유 조합 | 사용자가 원하는 조합으로 커스텀 구현 |
| COMBINED SAMPLE | 실제 사례 즉시 구현 | Case W에서 이 프레임 하나만 참조하여 전체 구현 |
