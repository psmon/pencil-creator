# v2.5.0 — Design Identity Overhaul

**날짜**: 2026-03-31
**유형**: 마이너 (에이전트+엔진 계층 정합성 업데이트)
**트리거**: 8회 평가 누적 후 agents/engine 계층이 옛 글쓰기 시스템 용어를 사용하고 있어 4-case 디자인 시스템과 불일치 발견

---

## 변경 사유

v2.0.0에서 글쓰기 하네스 → 디자인 하네스로 전환했으나, agents(`design-evaluator.md`)와 engine(`level-achievement-system.md`) 계층에 옛 시스템의 잔재가 남아 있었다:

- design-evaluator: "Case P|W"만 언급, Case A/B/C 미반영
- level-achievement-system: "5축 평가", "Type A~F", "content-evaluator" 등 글쓰기 용어 잔존
- RPG 칭호: "작가" 중심 → 디자인 프로젝트와 불일치
- 업적 카테고리: 글쓰기(AI/LLM, DevOps 등) → 디자인 카테고리 필요
- XP 유형배율: 옛 Type A~F → 현행 Case A/B/C/W ×1.2와 불일치
- 업적 상태: 8회 평가 진행되었으나 achievements.json이 2회분만 반영

---

## 변경 내역

### 1. design-evaluator.md (Layer 2: Agents)

| 항목 | Before | After |
|------|--------|-------|
| 케이스 목록 | Case P, W | Case A, B, C, W |
| 축 목록 | P1/P2/P3, W1/W2/W3 | A1~A3, B1~B3, C1~C3, W1~W3 |
| 출력 형식 | "Case {P\|W}" | "Case {A\|B\|C\|W}" |
| 로그 파일명 | case{P\|W} | case{A\|B\|C\|W} |
| 파이프라인 감지 | 없음 | 세션 내 + 당일 로그 기반 감지 로직 추가 |
| 업적 동기화 | 없음 | recording 단계에서 achievements.json 갱신 명시 |

### 2. level-achievement-system.md (Layer 3: Engine)

| 항목 | Before | After |
|------|--------|-------|
| 트리거 설명 | "content-evaluator가 5축 평가" | "design-evaluator가 3축 평가" |
| XP 공식 | 기본×등급×유형 | 기본×등급×케이스×파이프라인 (4요소 분리) |
| 유형배율 | Type A~F (×1.0~×2.0) | Case A/B/C/W (모두 ×1.2) |
| 파이프라인배율 | 없음 (별도 계산) | 공식에 통합 (×1.0~×1.5) |
| 레벨 칭호 | 작가 중심 (펜 잡은 모험가, 키보드 워리어 등) | 디자인 중심 (펜슬을 잡은 모험가, 프레임 워리어, 모션 마에스트로 등) |
| 만렙 칭호 | "BloomLabs의 신화" | "Pencil Creator의 신화" |
| 업적 카테고리 | 11개 글쓰기 도메인 (AI/LLM, DevOps 등) | 10개 디자인 도메인 (UI컴포넌트, 사이버/디지털 등) |
| 업적 패러디 | 글쓰기 주제 | 디자인/애니메이션 주제 |
| 반복 업적 기준 | 10/50/100회 | 5/15/30회 (디자인 작업은 글보다 무겁기 때문) |
| 특별 업적 | 7개 (글쓰기용) | 12개 (파이프라인, COMBINED SAMPLE 등 추가) |
| 출력 형식 | CONTENT QUEST | DESIGN QUEST + 파이프라인 보너스 표시 |
| 카테고리 판별 | 글쓰기 키워드 | 디자인 키워드 |

### 3. achievements.json — 상태 동기화

8회 평가 기록을 기반으로 누락된 업적을 소급 반영:
- 6개 카테고리 첫도전 업적 달성 기록
- 6개 특별 업적 달성 (펜슬을 잡은 순간, 넌 내게 A를 줬어, 콤보 개시!, 유혹을 이겨낸 자, A는 멈추지 않아, 합체 로봇)
- consecutive_a_grades: 0 → 7 (8회 중 첫 1회만 B, 이후 7연속 A)

### 4. status.json — 칭호 동기화

- "숙련 작가" → "숙련 디자이너"

### 5. harness.config.json — 버전 범프

- 2.4.0 → 2.5.0
