사용스킬 : harness-usage 

파일을 zip으로 업로드하면  해당 zip을 풀어 웹을 게시하고 관리하는
웹을 먼저 디자인하려고합니다. 웹페이지를 구성을참고 지침에의한 디자인

## 디자인지침
- projects/design/publisher-app.pen 을 이용해 먼저 디자인
  - wpf-animation 이펙트를 참고해 풍부한 애니메이션 효과를 줍니다.
  - 룩앤필 : 조금 화사한 느낌의 모던한 테마

## 웹페이지 구성
메뉴
- 대시보드 : 게시된 웹 등록현황
- 웹ZIP 업로드
  - 업로드시 프로그레스바 지원
- 웹 게시
  - 업로드된 파일 추가정보 입력후 게시함
  - 게시유효검사 : 웹페이지 이름(필수),소개(필수),게시자(필수),파비콘(옵션) : 기본유효검사(index.html 존재여부,html파일인지 기본체크(필수), jss/css/img 연결 깨짐체크 (해당파일필수아님) )
- 게시된 웹보기및 편집
  - 카드스타일로 게시된 웹보기 : 클릭시 새창으로 보기
  - 삭제가능

## 웹.zip 파일구성 예
- index.html
- css
- js
- img

## 후속조치
- 조사한 wpf를 참고안함 => 조사자료가 있으니 해당정보를 참고해 업데이트 유도
- 룩앤필이 밝음에도불구 애니메이션 정의가 어두움 그대로채택 => 애니메이션 효과를 룩앤필 기준으로 조정 지침

### 하네스 업데이트: 후속조치 적용을 위해 하네스 업데이트 진행 
스킬 : harness-creator
```
케이스A.
wpf-템플릿조사 후 템플릿보강을 하는 기능이 존재하며 이경우 xaml 을 직접조사해 wpf-animation.pen에 디자인을 확보하는 케이스이며
트리거참고 : wpf-템플릿조사 후 템플릿보강해

케이스B.
지금 진행된 케이스는
wpf-템플릿조사 업데이트가 아닌 , wpf-animation.pen 참고해, 프로젝트 요구사항을 파악해 새로운 펜슬을 디자인해 케이스임
트리거참고 : wpf-animation 이펙트를 참고해 OO 요구로 펜슬디자인요청

즉 A와 B의 하네스가 구분되어야하니 이부분을 참고 업데이트
케이스B는 현재 진행과정이 잘수행되었으니 방금 성공한 세션경험기반을 업데이트할것

이에따라 하네스 평가도 A,B를 구분할것
```
